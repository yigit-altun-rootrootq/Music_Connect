WHAT CAN IT DO?

🎛️

100% Blueprint Friendly

No C++ required. Drag, drop, and connect nodes. Works in any Blueprint Actor, GameMode, or PlayerController.

🌐

Multi-Platform Streaming

Search by song name or paste a URL from YouTube, Spotify, SoundCloud, or Apple Music — it just works.

⏯️

Full Playback Control

Play, Pause, Resume, Stop, Next/Previous Track, Jump To Track, Loop, Shuffle — all available as Blueprint nodes.

🔊

Volume & Fade

Set volume instantly or smoothly fade in/out over time. Range is 0.0–1.0, no extra setup needed.

📡

Event Dispatchers

React to state changes automatically — On State Changed, On Track Changed, On Error. Perfect for dynamic UIs.

📦

Packaged Game Ready

One extra step in Project Settings and your shipped game keeps streaming music without any issues.

// Quick Start

READY IN 60 SECONDS

01

Add the Component

Open your Blueprint Actor, go to the Components panel, click + Add, and search for Music Connect Player.

02

Use the Play Node

Drag your component into the Event Graph, pull a wire and search Play Music Stream. Type a song name or paste a URL into the Search Or URL pin.

03

Handle Events

Connect the On Playing pin to update your UI, On Error to show a fallback, and On Track Changed for playlists.

04

Package Your Game

In Project Settings, add Plugins/MusicConnect/Binaries/ThirdParty to Additional Non-Asset Directories to Copy. Done!

// Example Blueprint equivalent in pseudocode
Event BeginPlay
  → MusicConnectPlayer.PlayMusicStream("Daft Punk Get Lucky")
    On Playing → UpdateMusicUI()
    On Error → ShowErrorMessage()
